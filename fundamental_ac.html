Fundamental Access Control




Apa itu Access Control

Access control adalah proteksi layer pertama terhadap aset aset perusahaan atau organisasi, umumnya access control menggunakan beberapa layer teknologi. Namun secara teori ada yang dinamakan ada object access control dan subyek access control. Object adalah sebuah bentuk pasif dari aset perusahaan sedangkan subject adalah bentuk active dari perusahaan, contoh object adalah jaringan perusahaan, database perusahaan, email dll, sedangkan contoh subject adalah user atau aplikasi lain yang melakukan akses ke object tersebut.

Access Control menjaga agar informasi rahasia—seperti data pelanggan dan kekayaan intelektual—tidak dicuri oleh aktor jahat atau pengguna tidak sah lainnya. Ini juga mengurangi risiko eksfiltrasi data oleh karyawan dan mencegah ancaman berbasis web. Alih-alih mengelola izin secara manual, sebagian besar organisasi yang digerakkan oleh keamanan bersandar pada solusi Manajemen identitas dan akses untuk menerapkan kebijakan Access Control.

Berbagai Jenis Access Control

Discretionary Access Control (DAC)

Discretionary Access Constrol (DAC) merupakan suatu mekanisme data security yang menitik beratkan pada security objects, security subjects, dan access previleges dalam hal ini, DAC akan membatasi akses pada objek-objek berdasarkan identitas subjek atau group dimana mereka berada. Dalam DAC, keamanan data dapat direpresentasikan bahwa :

  1. User dapat menjaga data yang mereka miliki
  2. Owner dapat memberikan grant pada user lainnya
  3. Owner dapat memberikan definisi tipe akses yang akan diberikan pada user apakah akses tersebut berupa read, write, execute

Definisi Discretionary Access Control yaitu suatu mekanisme dalam pembatasan akses kepada objek-objeknya.

Roled-Based Access Control (RBAC)

Dalam RBAC, hak akses diberikan berdasarkan peran atau fungsi bisnis yang ditentukan. Tujuannya adalah untuk memberi pengguna hanya data yang mereka butuhkan untuk melakukan pekerjaan mereka—dan tidak lebih. Misalkan ruangan manajer hanya dapat diakses oleh manajer itu sendiri, sedangkan karyawan tidak dapat mengakses ruangan tersebut.

Contoh: pada MySQL setiap user diberikan peran tertentu untuk mengurus suatu database seperti mengunci database, mengupdate, atau juga user yang memiliki hak penuh di database.

Mandatory Access Control (MAC)

Dalam MAC, pengguna diberikan akses dalam bentuk izin dimana hak akses diberikan dan diatur oleh otoritas pusat dan mengaturnya dalam tingkatan, yang secara seragam memperluas cakupannya. Contoh: user ‘root’ MySQL dapat memberikan privileges tertentu kepada user lain untuk mengurus database tertentu sesuia dengan job yang diberikan si user ‘root’.

Mandatory Acces Control adalah salah satu sistem akses paling aman, karena cukup tahan gangguan. Berbeda dengan RBAC, pengguna tidak dapat membuat perubahan. Pemeriksaan dan penegakan hak akses sepenuhnya otomatis. Ini meminjamkan Kontrol Akses Wajib tingkat kerahasiaan yang tinggi. Selain itu, sistem ini menawarkan tingkat integritas yang tinggi: Data tidak dapat dimodifikasi tanpa otorisasi yang tepat dan dengan demikian dilindungi dari gangguan.

Attribute-Based Access Control (ABAC)

Dalam ABAC, hak akses diberikan secara fleksibel berdasarkan kombinasi atribut dan kondisi lingkungan, seperti waktu dan lokasi. ABAC adalah model kontol akses paling terperinci dan membantu mengurangi jumlah penetapan peran.

Contoh: di suatu database ada Atribute Editor, Security , dan Manajemen. Editor harus memilik akses mengubah, menghapus, dan membuat (ALTER, CREATE, DELETE), dan Security harus memilik hak untuk melihat semua proses dan mengunci table (SHOW PROCESSLIST & LOCK TABLES) , dan Manajemen memiliki akses untuk memberikan hak-hak yang berkaitan tadi (OPTION PRIVILEGES). Berdasarkan atribut-atribut tersebut admin/root dapat menerapkan kontrol akses terperinci berbasis kebijakan, menggunakan kombinasi atribut yang berbeda untuk menciptakan kondisi akses yang spesifik atau seluas yang diminta oleh situasi.
